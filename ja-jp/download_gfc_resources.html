グローバルファイルキャッシュエッジインスタンスの展開を開始する前に

2020年8月20日寄稿者 このページのPDFをダウンロード

リモートオフィスにGlobalFile Cache Edgeソフトウェアをインストールする前に、知っておく必要のある多くの要件があります。

必要なリソースをダウンロードする

ブランチオフィスで使用する予定のグローバルファイルキャッシュ仮想テンプレート、ソフトウェアインストールパッケージ、および追加のリファレンスドキュメントをダウンロードします。

  • Windows Server 2016仮想テンプレート：

    NetApp GFC（VMware VSphere 6.5以降）を含むWindows Server 2016 .OVA
    NetApp GFC（Microsoft Hyper-v）を含むWindows Server 2016 .VHDX

  • Windows Server 2019仮想テンプレート：

    NetApp GFC（VMware VSphere 6.5以降）を含むWindows Server 2019 .OVA
    NetApp GFC（Microsoft Hyper-v）を含むWindows Server 2019 .VHDX

  • グローバルファイルキャッシュエッジソフトウェア：

    NetApp GFCソフトウェア（.EXE）

  • グローバルファイルキャッシュのドキュメント：

    NetAppグローバルファイルキャッシュユーザーガイド

グローバルファイルキャッシュエッジの設計と展開

要件に応じて、ブランチオフィスでの同時ユーザーセッションに基づいて、1つまたは複数のグローバルファイルキャッシュエッジインスタンスを展開する必要がある場合があります。 Edgeインスタンスは、関連付けられたグローバルファイルキャッシュコアインスタンスから透過的に拡張された、ブランチオフィス内のエンドユーザーに仮想ファイル共有を提示します。グローバルファイルキャッシュエッジには、ブランチオフィス内のキャッシュファイルを含むD:\ NTFSボリュームが含まれている必要があります。

グローバルファイルキャッシュエッジの場合、 サイズ設定のガイドラインを理解することが重要です。これは、グローバルファイルキャッシュの展開に適した設計を行うのに役立ちます。また、規模、リソースの可用性、および冗長性の観点から、環境に適したものを決定する必要があります。

グローバルファイルキャッシュエッジインスタンス

グローバルファイルキャッシュエッジインスタンスを展開するときは、Windows Server .OVAまたはDatacenterEdition、Windows Server .OVAまたはDatacenterEditionを展開するか、グローバルファイルキャッシュの.OVAまたは.VHDテンプレートを使用して、単一のVMをプロビジョニングする必要があります。これには、選択したWindowsServerオペレーティングシステムとグローバルファイルキャッシュソフトウェアが含まれます。

クイックステップ

  1. グローバルファイルキャッシュ仮想テンプレート、Windows Server 2016 VM、またはWindows Server 2019StandardまたはDatacenterエディションを展開します。

  2. VMがネットワークに接続され、ドメインに参加し、RDPを介してアクセスできることを確認します。

  3. 最新のグローバルファイルキャッシュエッジソフトウェアをインストールします。

  4. グローバルファイルキャッシュ管理サーバーとコアインスタンスを特定します。

  5. グローバルファイルキャッシュエッジインスタンスを構成します。

グローバルファイルキャッシュエッジの要件

Global File Cache Edgeは、Windows Server 2016および2019をサポートするすべてのプラットフォームで機能するように設計されており、企業のリモートオフィスなどに簡素化されたITをもたらします。重要なことに、グローバルファイルキャッシュは、いくつかの基本レベルの要件を満たしていれば、ほとんどすべての場合に、既存のハードウェアインフラストラクチャ、仮想化、またはハイブリッド/パブリッククラウド環境に展開できます。

Global File Cache Edgeが最適に機能するには、次のハードウェアおよびソフトウェアリソースが必要です。全体的なサイズ設定ガイドラインの詳細については、サイズ設定ガイドラインを参照してください。

強化されたサーバーアプライアンス

グローバルファイルキャッシュインストールパッケージは、Microsoft WindowsServerインスタンス上に強化されたソフトウェアアプライアンスを作成します。グローバルファイルキャッシュパッケージをアンインストールしないでください。グローバルファイルキャッシュをアンインストールすると、サーバーインスタンスの機能に影響し、サーバーインスタンスの完全な再構築が必要になる場合があります。

物理的なハードウェア要件

  • 最小4つのCPUコア

  • 最小16GB RAM

  • 専用のシングルまたは冗長1Gbps NIC

  • 10k RPM SAS HDDまたはSSD（推奨）

  • ライトバックキャッシュ機能が有効になっているRAIDコントローラー

仮想展開の要件

ハイパーバイザープラットフォームは、ストレージサブシステムの観点からパフォーマンスが低下することが知られています（たとえば、遅延）。グローバルファイルキャッシュを使用して最適なパフォーマンスを得るには、SSDを備えた物理サーバーインスタンスをお勧めします。

仮想環境で最高のパフォーマンスを得るには、物理ホストの要件に加えて、次の要件とリソースの予約を満たす必要があります。

Microsoft Hyper-V 2012 R2以降：

  • プロセッサー（CPU）：CPUは静的として設定する必要があります：最小：4vCPUコア。

  • メモリ（RAM）：最小：静的として設定された16GB。

  • ハードディスクのプロビジョニング：ハードディスクは固定ディスクとして構成する必要があります。

VMware vSphere 6.x以降：

  • プロセッサー（CPU）：CPUサイクルの予約を設定する必要があります。最小：10000MHzで4つのvCPUコア。

  • メモリ（RAM）：最小：16GBの予約。

  • ハードディスクのプロビジョニング：

    • ディスクプロビジョニングは、シックプロビジョニングされた熱心なゼロとして設定する必要があります。

    • ハードディスク共有は高に設定する必要があります。

    • Microsoft Windowsがグローバルファイルキャッシュドライブをリムーバブルとして表示しないようにするには、vSphereClientを使用してDevices.hotplugをFalseに設定する必要があります。

  • ネットワーク：ネットワークインターフェイスをVMXNET3に設定する必要があります（VMツールが必要です）。

グローバルファイルキャッシュはWindowsServer 2016および2019で実行されるため、仮想化プラットフォームはオペレーティングシステムをサポートする必要があります。また、VMのゲストオペレーティングシステムのパフォーマンスとVMツールなどのVMの管理を強化するユーティリティとの統合も必要です。

パーティションのサイズ要件

  • C：\-最小250GB（システム/ブートボリューム）

  • D：\-最小1TB（グローバルファイルキャッシュインテリジェントファイルキャッシュ*用の個別のデータボリューム）

*最小サイズはアクティブなデータセットの2倍です。キャッシュボリューム（D：\）は拡張可能であり、Microsoft WindowsNTFSファイルシステムの制限によってのみ制限されます。

グローバルファイルキャッシュインテリジェントファイルキャッシュディスクの要件

グローバルファイルキャッシュインテリジェントファイルキャッシュディスク（D：\）のディスク遅延は、同時ユーザーあたり0.5ms未満の平均I / Oディスク遅延と1MiBpsスループットを提供する必要があります。

詳細については、「 NetAppグローバルファイルキャッシュユーザーガイド」を参照してください。

ネットワーキング

  • ファイアウォール：TCPポートは、グローバルファイルキャッシュエッジと管理サーバーおよびコアインスタンス間で許可する必要があります。

    グローバルファイルキャッシュTCPポート：443（HTTPS-LMS）、6618〜6630。

  • ネットワーク最適化デバイス（Riverbed Steelheadなど）は、グローバルファイルキャッシュ固有のポート（TCP 6618-6630）をパススルーするように構成する必要があります。

クライアントワークステーションとアプリケーションのベストプラクティス

グローバルファイルキャッシュは、顧客の環境に透過的に統合され、ユーザーがエンタープライズアプリケーションを実行しているクライアントワークステーションを使用して一元化されたデータにアクセスできるようにします。グローバルファイルキャッシュを使用すると、データはダイレクトドライブマッピングまたはDFS名前空間を介してアクセスされます。グローバルファイルキャッシュファブリック、インテリジェントファイルキャッシング、およびソフトウェアの主要な側面の詳細については、「グローバルファイルキャッシュの展開を開始する前に」セクションを参照してください。

最適なエクスペリエンスとパフォーマンスを確保するには、グローバルファイルキャッシュユーザーガイドに概説されているMicrosoftWindowsクライアントの要件とベストプラクティスに準拠することが重要です。これは、MicrosoftWindowsのすべてのバージョンに適用されます。

詳細については、「 NetAppグローバルファイルキャッシュユーザーガイド」を参照してください。

ファイアウォールとウイルス対策のベストプラクティス

グローバルファイルキャッシュは、最も一般的なウイルス対策アプリケーションスイートがグローバルファイルキャッシュと互換性があることを検証するために合理的な努力を払っていますが、ネットアップは、これらのプログラム、またはそれらに関連するアップデート、サービスパック、または変更。

グローバルファイルキャッシュは、グローバルファイルキャッシュが有効なインスタンス（コアまたはエッジ）に監視またはウイルス対策ソリューションをインストールまたは適用することを推奨していません。選択またはポリシーによってソリューションをインストールする場合は、次のベストプラクティスと推奨事項を適用する必要があります。一般的なウイルス対策スイートについては、 『 NetAppグローバルファイルキャッシュユーザーガイド』の付録Aを参照してください。

ファイアウォール設定

  • Microsoftファイアウォール：

    • ファイアウォール設定をデフォルトとして保持します。

    • 推奨事項：Microsoftファイアウォールの設定とサービスをデフォルト設定のOFFのままにし、標準のグローバルファイルキャッシュエッジインスタンスでは開始しないでください。

    • 推奨事項：Microsoftファイアウォールの設定とサービスをデフォルト設定のONのままにし、ドメインコントローラーの役割も実行するEdgeインスタンスに対して開始します。

  • 企業ファイアウォール：

    • グローバルファイルキャッシュコアインスタンスはTCPポート6618-6630でリッスンし、グローバルファイルキャッシュエッジインスタンスがこれらのTCPポートに接続できることを確認します。

    • グローバルファイルキャッシュインスタンスには、TCPポート443（HTTPS）でグローバルファイルキャッシュ管理サーバーとの通信が必要です。

  • ネットワーク最適化ソリューション/デバイスは、グローバルファイルキャッシュ固有のポートをパススルーするように構成する必要があります。

アンチウイルスのベストプラクティス

このセクションは、グローバルファイルキャッシュを実行しているWindowsServerインスタンスでウイルス対策ソフトウェアを実行する際の要件を理解するのに役立ちます。 Global File Cacheは、Cylance、McAfee、Symantec、Sophos、Trend Micro、Kaspersky、Windows Defenderなど、最も一般的に使用されているウイルス対策製品を、Global FileCacheと組み合わせて使用するためにテストしました。

Edgeアプライアンスにウイルス対策を追加すると、ユーザーのパフォーマンスに10〜20％の影響を与える可能性があります。

詳細については、「 NetAppグローバルファイルキャッシュユーザーガイド」を参照してください。

除外を構成する

ウイルス対策ソフトウェアまたはその他のサードパーティのインデックス作成またはスキャンユーティリティは、EdgeインスタンスのドライブD：\をスキャンしないでください。エッジサーバードライブD：\のこれらのスキャンにより、キャッシュ名前空間全体に対して多数のファイルオープン要求が発生します。これにより、WANを介したすべてのファイルサーバーへのファイルフェッチがデータセンターで最適化されます。 WAN接続のフラッディングとエッジインスタンスへの不要な負荷が発生し、パフォーマンスが低下します。

D：\ドライブに加えて、次のグローバルファイルキャッシュディレクトリとプロセスは、通常、すべてのウイルス対策アプリケーションから除外する必要があります。

  • C:\Program Files\TalonFAST\

  • C:\Program Files\TalonFAST\Bin\LMClientService.exe

  • C:\Program Files\TalonFAST\Bin\LMServerService.exe

  • C:\Program Files\TalonFAST\Bin\Optimus.exe

  • C:\Program Files\TalonFAST\Bin\tafsexport.exe

  • C:\Program Files\TalonFAST\Bin\tafsutils.exe

  • C:\Program Files\TalonFAST\Bin\tapp.exe

  • C:\Program Files\TalonFAST\Bin\tfs.exe

  • C:\Program Files\TalonFAST\Bin\TService.exe

  • C:\Program Files\TalonFAST\Bin\tum.exe

  • C:\Program Files\TalonFAST\FastDebugLogs\

  • C:\Windows\System32\drivers\tfast.sys

  • \\?\TafsMtPt:\ or \\?\TafsMtPt*

  • \Device\TalonCacheFS\

  • \\?\GLOBALROOT\Device\TalonCacheFS\

  • \\?\GLOBALROOT\Device\TalonCacheFS\*

NetAppサポートポリシー

グローバルファイルキャッシュインスタンスは、Windows Server2016および2019プラットフォームで実行されるプライマリアプリケーションとしてグローバルファイルキャッシュ用に特別に設計されています。グローバルファイルキャッシュは、ディスク、メモリ、ネットワークインターフェイスなどのプラットフォームリソースへの優先アクセスを必要とし、これらのリソースに高い要求を課す可能性があります。仮想展開には、メモリ/ CPUの予約と高性能ディスクが必要です。

  • グローバルファイルキャッシュのブランチオフィス展開の場合、グローバルファイルキャッシュを実行しているサーバーでサポートされるサービスとアプリケーションは次のように制限されます。

    • DNS / DHCP

    • Active Directoryドメインコントローラー（グローバルファイルキャッシュは別のボリュームにある必要があります）

    • 印刷サービス

    • Microsoft System Center Configuration Manager（SCCM）

    • グローバルファイルキャッシュで承認されたクライアント側のシステムエージェントとアンチウイルスアプリケーション

  • NetAppのサポートとメンテナンスは、グローバルファイルキャッシュにのみ適用されます。

  • データベースサーバーやメールサーバーなど、通常はリソースを大量に消費する基幹業務の生産性ソフトウェアはサポートされていません。

  • お客様は、グローバルファイルキャッシュを実行しているサーバーにインストールされる可能性のある非グローバルファイルキャッシュソフトウェアについて責任を負います。

    • サードパーティのソフトウェアパッケージによってソフトウェアまたはリソースがグローバルファイルキャッシュと競合したり、パフォーマンスが低下したりした場合、グローバルファイルキャッシュのサポート組織は、グローバルファイルキャッシュを実行しているサーバーからソフトウェアを無効にするか削除するようにお客様に要求する場合があります。

    • グローバルファイルキャッシュアプリケーションを実行しているサーバーに追加されたソフトウェアのすべてのインストール、統合、サポート、およびアップグレードについては、お客様の責任です。

  • ウイルス対策ツールやライセンスエージェントなどのシステム管理ユーティリティ/エージェントが共存できる場合があります。ただし、上記のサポートされているサービスとアプリケーションを除いて、これらのアプリケーションはグローバルファイルキャッシュではサポートされていないため、上記と同じガイドラインに従う必要があります。

    • 追加されたソフトウェアのすべてのインストール、統合、サポート、およびアップグレードについては、お客様の責任です。

    • お客様が、ソフトウェアまたはリソースがグローバルファイルキャッシュと競合する、または原因であると疑われるサードパーティのソフトウェアパッケージをインストールした場合、またはパフォーマンスが低下した場合、グローバルファイルキャッシュのサポート組織が無効化/削除する必要がある場合があります。ソフトウェア。
