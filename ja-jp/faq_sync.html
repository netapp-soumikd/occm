CloudSyncテクニカルFAQ

2020年10月2日寄稿者 このページのPDFをダウンロード

このFAQは、質問に対する簡単な回答を探している場合に役立ちます。

入門

次の質問は、クラウド同期の開始に関連しています。

Cloud Syncはどのように機能しますか？

Cloud Syncは、NetAppデータブローカーソフトウェアを使用して、ソースからターゲットにデータを同期します（これは同期関係と呼ばれます）。

データブローカーは、ソースとターゲット間の同期関係を制御します。同期関係を設定した後、Cloud Syncはソースシステムを分析し、それを複数のレプリケーションストリームに分割して、選択したターゲットデータにプッシュします。

最初のコピーの後、サービスは設定したスケジュールに基づいて変更されたデータを同期します。

14日間の無料トライアルはどのように機能しますか？

14日間の無料トライアルは、CloudSyncサービスにサインアップすると開始されます。 14日間作成したCloudSync関係のNetApp料金は発生しません。ただし、デプロイするデータブローカーのすべてのリソース料金は引き続き適用されます。

Cloud Syncの費用はいくらですか？

Cloud Syncの使用に関連するコストには、サービス料金とリソース料金の2種類があります。

サービス料

従量課金制の場合、Cloud Syncサービスの料金は、作成した同期関係の数に基づいて1時間ごとになります。

  • AWSで従量課金制を表示する

  • AWSで年間価格を表示する

  • Azureで価格を表示する

Cloud Syncライセンスは、ネットアップの担当者からも入手できます。各ライセンスにより、12か月間で20の同期関係が可能になります。

ライセンスの詳細をご覧ください。

リソース料金

リソース料金は、クラウドでデータブローカーを実行するためのコンピューティングとストレージのコストに関連しています。

Cloud Syncはどのように請求されますか？

14日間の無料トライアルが終了した後、同期関係の料金を支払う方法は2つあります。最初のオプションは、AWSまたはAzureからサブスクライブすることです。これにより、従量制または年払いが可能になります。 2番目のオプションは、NetAppから直接ライセンスを購入することです。

クラウドの外でCloudSyncを使用できますか？

はい、クラウド以外のアーキテクチャでCloudSyncを使用できます。ソースとターゲットはオンプレミスに存在でき、データブローカーもオンプレミスに存在できます。

クラウドの外部でCloudSyncを使用する場合の次の重要なポイントに注意してください。

  • オンプレミス同期の場合、プライベートAmazonS3バケットはNetAppStorageGRIDを介して利用できます。

  • データブローカーは、CloudSyncサービスと通信するためにインターネット接続を必要とします。

  • NetAppから直接ライセンスを購入しない場合は、PAYGO CloudSyncサービスの請求にAWSまたはAzureアカウントが必要になります。

Cloud Syncにアクセスするにはどうすればよいですか？

Cloud Syncは、 CloudManagerの[同期]タブから利用できます。

サポートされているソースとターゲット

以下の質問は、同期関係でサポートされているソースとターゲットに関連しています。

Cloud Syncはどのソースとターゲットをサポートしていますか？

Cloud Syncは、さまざまなタイプの同期関係をサポートしています。リスト全体を表示します。

Cloud SyncはどのバージョンのNFSとSMBをサポートしていますか？

Cloud Syncは、NFSバージョン3以降、およびSMBバージョン1以降をサポートします。

同期要件の詳細をご覧ください。

Amazon S3がターゲットの場合、データを特定のS3ストレージクラスに階層化できますか？

はい、AWS S3がターゲットの場合、特定のS3ストレージクラスを選択できます。

  • 標準（これはデフォルトのクラスです）

  • インテリジェントティアリング

  • 標準-アクセス頻度が低い

  • 1つのゾーン-アクセス頻度が低い

  • 氷河

  • グレイシャーディープアーカイブ

Azure Blobストレージのストレージ階層はどうですか？

Blobコンテナーがターゲットである場合、特定のAzureBlobストレージ層を選択できます。

  • ホットストレージ

  • 冷蔵

ネットワーキング

次の質問は、CloudSyncのネットワーク要件に関連しています。

Cloud Syncのネットワーク要件は何ですか？

Cloud Sync環境では、データブローカーが、選択したプロトコル（NFS、SMB、EFS）またはオブジェクトストレージAPI（Amazon S3、Azure Blob、IBM Cloud Object Storage）を介してソースおよびターゲットに接続されている必要があります。

さらに、データブローカーは、Cloud Syncサービスと通信し、他のいくつかのサービスやリポジトリに接続できるように、ポート443を介したアウトバウンドインターネット接続を必要とします。

詳細については、ネットワーク要件を確認してください。

データブローカーの接続に関連するネットワークの制限はありますか？

データブローカーにはインターネットアクセスが必要です。 AzureまたはGoogleCloud Platformにデータブローカーをデプロイする場合、プロキシサーバーはサポートされません。

データの同期

次の質問は、データ同期がどのように機能するかに関するものです。

同期はどのくらいの頻度で発生しますか？

デフォルトのスケジュールは、毎日の同期に設定されています。最初の同期後、次のことができます。

  • 同期スケジュールを希望の日数、時間、または分に変更します

  • 同期スケジュールを無効にする

  • 同期スケジュールを削除します（データが失われることはなく、同期関係のみが削除されます）

最小同期スケジュールは何ですか？

1分ごとにデータを同期するように関係をスケジュールできます。

ファイルの同期に失敗した場合、データブローカーは再試行しますか？それともタイムアウトしますか？

単一のファイルの転送に失敗しても、データブローカーはタイムアウトしません。代わりに、データブローカーはファイルをスキップする前に3回再試行します。再試行値は、同期関係の設定で構成できます。

同期関係の設定を変更する方法を学びます。

非常に大きなデータセットがある場合はどうなりますか？

1つのディレクトリに600,000以上のファイルが含まれている場合は、ペイロードを処理するようにデータブローカーを構成できるように、お問い合わせください。データブローカーマシンにメモリを追加する必要があるかもしれません。

セキュリティ

セキュリティに関連する次の質問。

Cloud Syncは安全ですか？

はい。すべてのCloudSyncサービスのネットワーク接続は、 Amazon Simple Queue Service（SQS）を使用して行われます。

データブローカーとAmazonS3、Azure Blob、Google Cloud Storage、IBM Cloud Object Storageの間のすべての通信は、HTTPSプロトコルを介して行われます。

オンプレミス（ソースまたは宛先）システムでCloud Syncを使用している場合、推奨される接続オプションは次のとおりです。

  • AWS Direct Connect、Azure ExpressRoute、またはGoogle Cloud Interconnect接続。これは、インターネット以外でルーティングされます（指定したクラウドネットワークとのみ通信できます）。

  • オンプレミスゲートウェイデバイスとクラウドネットワーク間のVPN接続

  • S3バケット、Azure Blobストレージ、またはGoogle Cloud Storageを使用した安全性の高いデータ転送のために、AmazonプライベートS3エンドポイント、Azure仮想ネットワークサービスエンドポイント、またはプライベートGoogleアクセスを確立できます。

これらの方法はいずれも、オンプレミスNASサーバーとCloudSyncデータブローカー間の安全な接続を確立します。

データはCloudSyncによって暗号化されていますか？

  • Cloud Syncは、ソースNFSサーバーとターゲットNFSサーバー間のデータインフライト暗号化をサポートします。詳細をご覧ください。

  • 暗号化はSMBではサポートされていません。

  • Amazon S3バケットが同期関係のターゲットである場合、AWSKMS暗号化またはAES-256暗号化を使用してデータ暗号化を有効にするかどうかを選択できます。

権限

次の質問は、データのアクセス許可に関連しています。

SMBデータのアクセス許可はターゲットの場所に同期されていますか？

Cloud Syncを設定して、ソースSMB共有とターゲットSMB共有の間のアクセス制御リスト（ACL）を保持できます。または、ACLを自分で手動でコピーすることもできます。 SMB共有間でACLをコピーする方法を学びます。

NFSデータのアクセス許可はターゲットの場所に同期されていますか？

Cloud Syncは、次のようにNFSサーバー間でNFSアクセス許可を自動的にコピーします。

  • NFSバージョン3：Cloud Syncは、アクセス許可とユーザーグループの所有者をコピーします。

  • NFSバージョン4：CloudSyncはACLをコピーします。

パフォーマンス

次の質問は、CloudSyncのパフォーマンスに関連しています。

同期関係の進行状況インジケーターは何を表していますか？

同期関係は、データブローカーのネットワークアダプターのスループットを示します。複数のデータブローカーを使用して同期パフォーマンスを高速化した場合、スループットはすべてのトラフィックの合計になります。このスループットは20秒ごとに更新されます。

パフォーマンスの問題が発生しています。同時転送の数を制限できますか？

データブローカーは一度に4つのファイルを同期できます。非常に大きなファイル（それぞれ複数のTB）がある場合、転送プロセスの完了に長い時間がかかる可能性があり、パフォーマンスに影響を与える可能性があります。

同時転送の数を制限すると役立つ場合があります。ヘルプが必要な場合はお問い合わせください。

Azure NetAppファイルでパフォーマンスが低下するのはなぜですか？

Azure NetAppファイルとの間でデータを同期する場合、ディスクサービスレベルが標準の場合、障害やパフォーマンスの問題が発生する可能性があります。

同期パフォーマンスを向上させるには、サービスレベルをプレミアムまたはウルトラに変更します。

Azure NetAppFilesのサービスレベルとスループットの詳細をご覧ください。

Cloud Volumes Service for AWSでパフォーマンスが低下するのはなぜですか？

クラウドボリュームとの間でデータを同期する場合、クラウドボリュームのパフォーマンスレベルが標準の場合、障害やパフォーマンスの問題が発生する可能性があります。

同期パフォーマンスを向上させるには、サービスレベルをプレミアムまたはエクストリームに変更します。

いくつのデータブローカーが必要ですか？

新しい関係を作成するときは、単一のデータブローカーから開始します（高速同期関係に属する既存のデータブローカーを選択した場合を除く）。多くの場合、単一のデータブローカーが同期関係のパフォーマンス要件を満たすことができます。そうでない場合は、データブローカーを追加することで同期パフォーマンスを加速できます。ただし、最初に同期パフォーマンスに影響を与える可能性のある他の要因を確認する必要があります。

複数の要因がデータ転送のパフォーマンスに影響を与える可能性があります。全体的な同期パフォーマンスは、ネットワーク帯域幅、遅延、ネットワークトポロジ、およびデータブローカーのVM仕様とストレージシステムのパフォーマンスによって影響を受ける可能性があります。たとえば、同期関係にある単一のデータブローカーは100 MB / sに達する可能性がありますが、ターゲットのディスクスループットでは64 MB / sしか許可されない場合があります。その結果、データブローカーはデータのコピーを試み続けますが、ターゲットはデータブローカーのパフォーマンスを満たすことができません。

したがって、ネットワークのパフォーマンスとターゲットのディスクスループットを必ず確認してください。

次に、データブローカーを追加してその関係の負荷を共有することにより、同期パフォーマンスを加速することを検討できます。同期パフォーマンスを加速する方法を学びます。

ものを削除する

次の質問は、ソースとターゲットからの同期関係とデータの削除に関連しています。

Cloud Syncの関係を削除するとどうなりますか？

関係を削除すると、今後のすべてのデータ同期が停止し、支払いが終了します。ターゲットに同期されたデータはすべてそのままです。

ソースサーバーから何かを削除するとどうなりますか？ターゲットからも削除されていますか？

デフォルトでは、アクティブな同期関係がある場合、ソースサーバーで削除されたアイテムは、次の同期時にターゲットから削除されません。ただし、各関係の同期設定にはオプションがあり、ソースから削除された場合にCloudSyncがターゲットの場所にあるファイルを削除するように定義できます。

同期関係の設定を変更する方法を学びます。

ターゲットから何かを削除するとどうなりますか？ソースからも削除されていますか？

アイテムがターゲットから削除された場合、そのアイテムはソースから削除されません。関係は一方向です—ソースからターゲットへ。次の同期サイクルで、Cloud Syncはソースとターゲットを比較し、アイテムが欠落していることを識別し、CloudSyncはそれをソースからターゲットに再度コピーします。

データブローカーの詳細

次の質問は、データブローカーに関するものです。

データブローカーのアーキテクチャについて説明していただけますか？

承知しました。最も重要なポイントは次のとおりです。

  • データブローカーは、Linuxホストで実行されるnode.jsアプリケーションです。

  • Cloud Syncは、データブローカーを次のようにデプロイします。

    • AWS：AWSCloudFormationテンプレートから

    • Azure：Azure ResourceManagerから

    • Google：Google Cloud DeploymentManagerから

    • 独自のLinuxホストを使用する場合は、ソフトウェアを手動でインストールする必要があります

  • データブローカーソフトウェアは、自動的に最新バージョンにアップグレードします。

  • データブローカーは、AWS SQSを信頼性の高い安全な通信チャネルとして、また制御と監視に使用します。 SQSは永続性レイヤーも提供します。

  • リレーションシップにデータブローカーを追加して、転送速度を上げ、高可用性を追加できます。 1つのデータブローカーに障害が発生した場合、サービスの回復力があります。
