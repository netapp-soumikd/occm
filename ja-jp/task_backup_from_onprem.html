オンプレミスのONTAPシステムからクラウドへのデータのバックアップ

2020年11月2日寄稿者 このページのPDFをダウンロード

オンプレミスONTAPシステムからクラウド内の低コストのオブジェクトストレージへのデータのバックアップを開始するには、いくつかの手順を実行します。

クイックスタート

これらの手順に従ってすぐに開始するか、残りのセクションまでスクロールして詳細を確認してください。

構成のサポートを確認する

  • オンプレミスクラスターを検出し、CloudManagerの作業環境に追加しました。詳細については、 ONTAPクラスタの検出を参照してください。

    • クラスタはONTAP9.7P5以降を実行しています。

    • クラスターにはSnapMirrorライセンスがあります。これはPREMまたはデータ保護バンドルの一部として含まれています。

  • Azure NetApp Cloud Manager Marketplaceオファリング、 AWS Cloud Manager Marketplaceオファリングをサブスクライブしているか、NetAppからBackup to CloudBYOLライセンスを購入してアクティブ化しました。

  • バックアップが配置されるストレージスペースの有効なクラウドプロバイダーサブスクリプションがあります。

  • AWSの場合、ONTAPクラスターがデータをS3にバックアップできるように、アクセスキーと必要なアクセス許可を持つアカウントが必要です。

システムでクラウドへのバックアップを有効にする

作業環境を選択し、右側のパネルの[クラウドへのバックアップ]サービスの横にある[アクティブ化]をクリックして、セットアップウィザードに従います。

クラウドプロバイダーを選択し、プロバイダーの詳細を入力します

プロバイダーを選択し、プロバイダーの詳細を入力します。また、ボリュームが存在するONTAPクラスター内のIPspaceを指定する必要があります。

バックアップポリシーを定義する

デフォルトのポリシーは、ボリュームを毎日バックアップし、各ボリュームの最新の30個のバックアップコピーを保持します。毎週または毎月のバックアップに変更するか、より多くのオプションを提供するシステム定義のポリシーの1つを選択します。

バックアップするボリュームを選択します

クラスタからバックアップするボリュームを特定します。

要件

オンプレミスボリュームのオブジェクトストレージへのバックアップを開始する前に、次の要件を読んで、サポートされている構成があることを確認してください。

ONTAPの要件

ONTAP9.7P5以降。

SnapMirrorライセンス（PREMまたはデータ保護バンドルの一部として含まれています）。

クラスターネットワークの要件

バックアップするボリュームをホストする各ONTAPノードには、クラスタ間LIFが必要です。 LIFは、ONTAPがオブジェクトストレージに接続するために使用するIPspaceに関連付けられている必要があります。管理SVMはIPspaceに存在する必要があります。 IPspacesの詳細をご覧ください。

クラウドへのバックアップを設定すると、使用するIPspaceの入力を求められます。各LIFが関連付けられているIPspaceを選択する必要があります。これは、「デフォルト」のIPスペースまたは作成したカスタムIPスペースである可能性があります。

サポートされている地域

オンプレミスシステムからのバックアップは、Cloud VolumesONTAPがサポートされているすべてのリージョンでサポートされています。

  • Azureの場合、サービスのセットアップ時にバックアップを保存するリージョンを指定します。

  • AWSの場合、バックアップはCloudManagerがインストールされているリージョンに保存されます。

ライセンス要件

クラウドPAYGOライセンスへのバックアップについては、サブスクリプションのAzure Marketplaceのクラウドマネージャのバックアップ製品またはAWS Marketplaceのクラウドマネージャのバックアップ製品は、あなたがクラウドへのバックアップ有効にする前に必要です。クラウドへのバックアップの請求は、このサブスクリプションを通じて行われます。

Backup to Cloud BYOLライセンスの場合、ライセンスの期間と容量の間サービスを使用できるようにするNetAppのシリアル番号が必要です。 バックアップBYOLライセンスの追加と更新を参照してください。

また、バックアップを配置するストレージスペースには、MicrosoftAzureまたはAmazonAWSのサブスクリプションが必要です。

AmazonS3の準備

Amazon S3を使用する場合は、Cloud ManagerがS3バケットにアクセスするためのアクセス許可を構成する必要があります。また、オンプレミスのONTAPクラスターがS3バケットにアクセスできるようにアクセス許可を構成する必要があります。

ステップ

  1. 次のS3アクセス許可（最新のCloud Managerポリシーから）を、CloudManagerにアクセス許可を提供するIAMロールに提供します。

     { "Sid": "backupPolicy", "Effect": "Allow", "Action": [ "s3:DeleteBucket", "s3:GetLifecycleConfiguration", "s3:PutLifecycleConfiguration", "s3:PutBucketTagging", "s3:ListBucketVersions", "s3:GetObject", "s3:ListBucket", "s3:ListAllMyBuckets", "s3:GetBucketTagging", "s3:GetBucketLocation", "s3:GetBucketPolicyStatus", "s3:GetBucketPublicAccessBlock", "s3:GetBucketAcl", "s3:GetBucketPolicy", "s3:PutBucketPublicAccessBlock" ], "Resource": [ "arn:aws:s3:::netapp-backup-*" ] },

  2. ONTAPクラスタがデータをS3にバックアップできるように、IAMユーザーに次の権限を付与します。

     "s3:ListAllMyBuckets", "s3:ListBucket", "s3:GetBucketLocation", "s3:GetObject", "s3:PutObject", "s3:DeleteObject"

    詳細については、 AWSドキュメント：IAMユーザーにパーミッションを委任するためのロールの作成を参照してください。

  3. アクセスキーを作成または検索します。

    Backup to Cloudは、アクセスキーをONTAPクラスターに渡します。資格情報は、Backup toCloudサービスに保存されません。

    詳細については、 AWSドキュメント：IAMユーザーのアクセスキーの管理を参照してください。

クラウドへのバックアップの有効化

作業環境から直接、いつでもクラウドへのバックアップを有効にします。

ステップ

  1. 作業環境を選択し、右側のパネルの[クラウドへのバックアップ]サービスの横にある[アクティブ化]をクリックします。

  2. プロバイダーを選択し、プロバイダーの詳細を入力します。

    • Azureの場合は、次のように入力します。

      1. バックアップに使用されるAzureサブスクリプションと、バックアップが保存されるAzureリージョン。

      2. リソースグループ-新しいリソースグループを作成するか、既存のリソースグループを選択できます。

      3. バックアップするボリュームが存在するONTAPクラスター内のIPspace。

    • AWSの場合、次のように入力します。

      1. バックアップの保存に使用されるAWSアクセスキーとシークレットキー。

      2. バックアップするボリュームが存在するONTAPクラスター内のIPspace。

        サービス開始後は、この情報を変更することはできませんのでご注意ください。

  3. 次に、[続行]をクリックします。

  4. [ポリシーの定義]ページで、バックアップスケジュールと保持値を選択し、[続行]をクリックします。

    既存のポリシーのリストを参照してください。

  5. バックアップするボリュームを選択し、[保存]をクリックします。

結果

Backup to Cloudは、選択した各ボリュームの初期バックアップの作成を開始します。

次は何ですか？

バックアップスケジュールの変更、ボリュームの復元などにより、バックアップを管理できます。
