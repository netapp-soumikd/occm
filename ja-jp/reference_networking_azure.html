AzureでCloudVolumesONTAPをデプロイおよび管理するためのネットワーク要件

2020年11月4日寄稿者このページのPDFをダウンロード

Cloud VolumesONTAPシステムが正しく動作できるようにAzureネットワークをセットアップします。これには、ConnectorおよびCloud VolumesONTAPのネットワークが含まれます。

Cloud VolumesONTAPの要件

Azureでは、次のネットワーク要件を満たす必要があります。

Cloud VolumesONTAPのアウトバウンドインターネットアクセス

Cloud Volumes ONTAPでは、ストレージの状態をプロアクティブに監視するNetApp AutoSupportにメッセージを送信するために、アウトバウンドインターネットアクセスが必要です。

Cloud Volumes ONTAPがAutoSupportメッセージを送信できるように、ルーティングおよびファイアウォールポリシーは、次のエンドポイントへのHTTP / HTTPSトラフィックを許可する必要があります。

  • https://support.netapp.com/aods/asupmessage

  • https://support.netapp.com/asupprod/post/1.0/postAsup

    AutoSupportを構成する方法を学びます。

セキュリティグループ

Cloud Managerがセキュリティグループを作成するため、セキュリティグループを作成する必要はありません。独自のルールを使用する必要がある場合は、以下のセキュリティグループルールを参照してください。

IPアドレスの数

Cloud Managerは、次の数のIPアドレスをAzureのCloud VolumesONTAPに割り当てます。

  • シングルノード：5つのIPアドレス

  • HAペア：16個のIPアドレス

    Cloud ManagerはHAペアでSVM管理LIFを作成しますが、Azureの単一ノードシステムでは作成しないことに注意してください。

    LIFは、物理ポートに関連付けられたIPアドレスです。 SnapCenterなどの管理ツールにはSVM管理LIFが必要です。
データ階層化のためのCloudVolumesonTAPからAzureBlobストレージへの接続

コールドデータをAzureBlobストレージに階層化する場合、Cloud Managerに必要なアクセス許可がある限り、パフォーマンス層と容量層の間に接続を設定する必要はありません。 Cloud Managerポリシーに次の権限がある場合、CloudManagerはVNetサービスエンドポイントを有効にします。

 "Microsoft.Network/virtualNetworks/subnets/write", "Microsoft.Network/routeTables/join/action",

これらの権限は、最新のCloudManagerポリシーに含まれています。

データ階層化の設定の詳細については、コールドデータを低コストのオブジェクトストレージに階層化するを参照してください。

他のネットワークのONTAPシステムへの接続

AzureのCloudVolumes ONTAPシステムと他のネットワークのONTAPシステム間でデータをレプリケートするには、Azure VNetと他のネットワーク（AWS VPCや企業ネットワークなど）との間にVPN接続が必要です。

手順については、 Microsoft Azureのドキュメント：Azureポータルでサイト間接続を作成するを参照してください。

コネクタの要件

コネクタがパブリッククラウド環境内のリソースとプロセスを管理できるようにネットワークを設定します。最も重要なステップは、さまざまなエンドポイントへのアウトバウンドインターネットアクセスを確保することです。

ネットワークがインターネットへのすべての通信にプロキシサーバーを使用している場合は、[設定]ページからプロキシサーバーを指定できます。プロキシサーバーを使用するためのコネクタの設定を参照してください。

ターゲットネットワークへの接続

コネクタには、Cloud VolumesONTAPをデプロイするVPCおよびVNetへのネットワーク接続が必要です。

たとえば、企業ネットワークにコネクタをインストールする場合は、Cloud VolumesONTAPを起動するVPCまたはVNetへのVPN接続を設定する必要があります。

アウトバウンドインターネットアクセス

コネクターは、パブリッククラウド環境内のリソースとプロセスを管理するためにアウトバウンドインターネットアクセスを必要とします。コネクタは、Azureでリソースを管理するときに、次のエンドポイントに接続します。
