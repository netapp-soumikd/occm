同期関係の管理

2020年10月2日寄稿者 このページのPDFをダウンロード

データをすぐに同期したり、スケジュールを変更したりすることで、いつでも同期関係を管理できます。

即時データ同期の実行

次にスケジュールされた同期を待つのではなく、ボタンを押すと、ソースとターゲットの間でデータをすぐに同期できます。

ステップ

  1. 同期ダッシュボードから、同期関係にカーソルを合わせ、アクションメニューをクリックします。

  2. [今すぐ同期]をクリックし、[同期]をクリックして確認します。

結果

Cloud Syncは、関係のデータ同期プロセスを開始します。

同期パフォーマンスの加速

関係にデータブローカーを追加することにより、同期関係のパフォーマンスを加速します。追加のデータブローカーは、新しいデータブローカーである必要があります。

これがどのように機能するか

リレーションシップ内の既存のデータブローカーが他の同期リレーションシップで使用されている場合、CloudSyncはそれらのリレーションシップにも新しいデータブローカーを自動的に追加します。

たとえば、次の3つの関係があるとします。

  • 関係1はデータブローカーAを使用します

  • 関係2はデータブローカーBを使用します

  • 関係3はデータブローカーAを使用します

リレーションシップ1のパフォーマンスを高速化して、そのリレーションシップに新しいデータブローカーを追加します（データブローカーC）。データブローカーAはリレーションシップ3でも使用されるため、新しいデータブローカーもリレーションシップ3に自動的に追加されます。

ステップ

  1. 関係にある既存のデータブローカーの少なくとも1つがオンラインであることを確認してください。

  2. 同期関係にカーソルを合わせ、アクションメニューをクリックします。

  3. [加速]をクリックします。

  4. プロンプトに従って、新しいデータブローカーを作成します。

結果

Cloud Syncは、新しいデータブローカーを同期関係に追加します。次のデータ同期のパフォーマンスを加速する必要があります。

同期関係の設定を変更する

ソースファイルとフォルダーをターゲットの場所で同期および維持する方法を定義する設定を変更します。

  1. 同期関係にカーソルを合わせ、アクションメニューをクリックします。

  2. [設定]をクリックします。

  3. 設定のいずれかを変更します。

    各設定の簡単な説明は次のとおりです。

    スケジュール

    将来の同期のために定期的なスケジュールを選択するか、同期スケジュールをオフにします。 1分ごとにデータを同期するように関係をスケジュールできます。

    再試行

    Cloud Syncがファイルをスキップする前に、ファイルの同期を再試行する回数を定義します。

    最近変更されたファイル

    スケジュールされた同期の前に最近変更されたファイルを除外することを選択します。

    ソース上のファイルを削除する

    Cloud Syncがファイルをターゲットの場所にコピーした後、ソースの場所からファイルを削除することを選択します。このオプションには、ソースファイルがコピーされた後に削除されるため、データが失われるリスクが含まれます。

    このオプションを有効にする場合は、データブローカーのlocal.jsonファイルのパラメーターも変更する必要があります。ファイルを開き、 workers.transferrer.delete-on-sourceという名前のパラメーターをtrueに変更します。

    ターゲット上のファイルを削除する

    ファイルがソースから削除された場合は、ターゲットの場所からファイルを削除することを選択します。デフォルトでは、ターゲットの場所からファイルを削除することはありません。

    オブジェクトのタグ付け

    AWS S3が同期関係のターゲットである場合、CloudSyncは同期操作に関連するメタデータでS3オブジェクトにタグを付けます。ご使用の環境で不要な場合は、S3オブジェクトのタグ付けを無効にすることができます。タグ付けを無効にしても、CloudSyncに影響はありません。CloudSyncは同期メタデータを別の方法で保存するだけです。

    ファイルタイプ

    各同期に含めるファイルタイプ（ファイル、ディレクトリ、シンボリックリンク）を定義します。

    ファイル拡張子を除外する

    ファイル拡張子を入力してEnterキーを押し、同期から除外するファイル拡張子を指定します。たとえば、 logまたは.logと入力して、*。 logファイルを除外します。複数の拡張子にはセパレータは必要ありません。次のビデオは短いデモを提供します：

    ファイルサイズ

    サイズに関係なくすべてのファイルを同期するか、特定のサイズ範囲にあるファイルのみを同期するかを選択します。

    日付が変更されました

    最終変更日、特定の日付の後、特定の日付より前、または時間範囲の間に変更されたファイルに関係なく、すべてのファイルを選択します。

    アクセス制御リストをターゲットにコピーします

    ソースSMB共有とターゲットSMB共有の間でアクセス制御リスト（ACL）をコピーすることを選択します。このオプションは、2020年2月23日のリリース以降に作成された同期関係でのみ使用できることに注意してください。

  4. [設定を保存]をクリックします。

結果

Cloud Syncは、新しい設定で同期関係を変更します。

関係の削除

ソースとターゲット間でデータを同期する必要がなくなった場合は、同期関係を削除できます。このアクションは、データブローカーインスタンスを削除せず、ターゲットからデータを削除しません。

ステップ

  1. 同期関係にカーソルを合わせ、アクションメニューをクリックします。

  2. [削除]をクリックし、もう一度[削除]をクリックして確認します。

結果

Cloud Syncは、同期関係を削除します。
