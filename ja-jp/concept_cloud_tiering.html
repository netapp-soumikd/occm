クラウド階層化について学ぶ

2020年10月20日寄稿者 このページのPDFをダウンロード

NetAppのクラウド階層化サービスは、非アクティブなデータをオンプレミスのONTAPクラスターからオブジェクトストレージに自動的に階層化することで、データセンターをクラウドに拡張します。これにより、アプリケーション層に変更を加えることなく、クラスター上の貴重なスペースをより多くのワークロードに解放できます。クラウド階層化により、データセンターのコストを削減し、CAPEXモデルからOPEXモデルへの切り替えが可能になります。

クラウド階層化サービスは、 FabricPoolの機能を活用します。 FabricPoolは、低コストのオブジェクトストレージへのデータの自動階層化を可能にするNetApp DataFabricテクノロジーです。アクティブなデータは高性能SSDに残りますが、非アクティブなデータはONTAPデータの効率を維持しながら、低コストのオブジェクトストレージに階層化されます。

特徴

クラウド階層化は、自動化、監視、レポート、および共通の管理インターフェイスを提供します。

  • 自動化により、オンプレミスのONTAPクラスターからクラウドへのデータ階層化のセットアップと管理が容易になります

  • 1枚のガラス板で、複数のクラスター間でFabricPoolを個別に管理する必要がなくなります。

  • レポートには、各クラスターのアクティブデータと非アクティブデータの量が表示されます

  • ヘルスステータスを階層化すると、問題が発生したときにそれを特定して修正するのに役立ちます

  • Cloud Volumes ONTAPシステムを使用している場合は、クラスターダッシュボードに表示されるため、ハイブリッドクラウドインフラストラクチャのデータ階層化の全体像を把握できます。

    Cloud Volumes ONTAPシステムは、CloudTieringから読み取り専用です。 CloudManagerの作業環境からCloudVolumesONTAPの階層化を設定します。

クラウド階層化が提供する価値の詳細については、 NetApp CloudCentralのクラウド階層化ページをご覧ください。

クラウド階層化はストレージのフットプリントを大幅に削減できますが、バックアップソリューションではありません。

サポートされているオブジェクトストレージプロバイダー

非アクティブなデータをONTAPクラスターからAmazonS3、Microsoft Azure Blobストレージ、Google Cloud Storage、またはStorageGRID（プライベートクラウド）に階層化できます。

価格とライセンス

従量課金制のサブスクリプション、 FabricPoolと呼ばれるONTAP階層化ライセンス、またはその両方を組み合わせて、クラウド階層化の料金を支払います。ライセンスをお持ちでない場合は、最初のクラスターで30日間の無料トライアルを利用できます。

データをStorageGRIDに階層化する場合、料金はかかりません。 BYOLライセンスもPAYGO登録も必要ありません。

価格の詳細を表示します。

30日間の無料トライアル

FabricPoolライセンスをお持ちでない場合は、最初のクラスターに階層化を設定すると、クラウド階層化の30日間の無料トライアルが開始されます。 30日間の無料トライアルが終了した後、従量課金制のサブスクリプション、FabricPoolライセンス、またはその両方を組み合わせてクラウド階層化の料金を支払う必要があります。

無料試用期間が終了し、ライセンスをサブスクライブまたは追加していない場合、ONTAPはコールドデータをオブジェクトストレージに階層化しなくなりますが、既存のデータには引き続きアクセスできます。

従量課金制のサブスクリプション

クラウド階層化は、従量課金モデルで消費ベースのライセンスを提供します。クラウドプロバイダーのマーケットプレイスでサブスクライブした後、階層化されたデータに対してGBごとに支払います。前払いはありません。毎月の請求を通じて、クラウドプロバイダーから請求されます。

無料トライアルがある場合、または独自のライセンス（BYOL）を持参している場合でも、サブスクライブする必要があります。

  • 購読すると、無料トライアルの終了後にサービスが中断することはありません。

    試用期間が終了すると、階層化したデータの量に応じて1時間ごとに課金されます。

  • FabricPoolライセンスで許可されているよりも多くのデータを階層化する場合、データの階層化は従量課金制のサブスクリプションを通じて続行されます。

    たとえば、10 TBのライセンスをお持ちの場合、10 TBを超えるすべての容量は、従量課金制のサブスクリプションを通じて課金されます。

無料試用期間中、またはFabricPoolライセンスを超えていない場合は、従量課金制のサブスクリプションから請求されることはありません。

従量課金制のサブスクリプションを設定する方法を学びます。

自分のライセンスを持参してください

NetAppからONTAPFabricPoolライセンスを購入して、独自のライセンスを持参してください。期間ベースまたは永久ライセンスを購入できます。

FabricPoolライセンスを購入したら、それをクラスターに追加する必要があります。これは、クラウド階層化から直接実行できます。

クラウド階層化を介してライセンスをアクティブ化した後、後で追加のアドオン容量を購入すると、クラスター上のライセンスが新しい容量で自動的に更新されます。新しいNetAppライセンスファイル（NLF）をクラスターに適用する必要はありません。

上記のように、クラスターにBYOLライセンスがある場合でも、従量課金制のサブスクリプションを設定することをお勧めします。

ライセンスを購入するには、お問い合わせください。

クラウド階層化の仕組み

クラウド階層化は、FabricPoolテクノロジーを使用して、オンプレミスONTAPクラスターからパブリッククラウドまたはプライベートクラウドのオブジェクトストレージに非アクティブな（コールド）データを自動的に階層化するNetAppマネージドサービスです。 ONTAPへの接続はコネクタから行われます。

次の画像は、各コンポーネント間の関係を示しています。

大まかに言えば、クラウド階層化は次のように機能します。

  1. CloudManagerからオンプレミスクラスターを検出します。

  2. バケット/コンテナやストレージクラスまたはアクセス階層など、オブジェクトストレージに関する詳細を提供することで階層を設定します。

  3. Cloud Managerは、オブジェクトストレージプロバイダーを使用するようにONTAPを構成し、クラスター上のアクティブデータと非アクティブデータの量を検出します。

  4. 階層化するボリュームと、それらのボリュームに適用する階層化ポリシーを選択します。

  5. ONTAPは、データが非アクティブと見なされるしきい値に達するとすぐに、非アクティブなデータのオブジェクトストアへの階層化を開始します（ボリューム階層化ポリシーを参照）。

オブジェクトストレージ

各ONTAPクラスタは、非アクティブなデータを単一のオブジェクトストアに階層化します。データ階層化を設定するときは、ストレージクラスまたはアクセス階層とともに、新しいバケット/コンテナを追加するか、既存のバケット/コンテナを選択するかを選択できます。

  • サポートされているS3ストレージクラスについて学ぶ

  • サポートされているAzureBlobアクセス階層について学ぶ

  • サポートされているGoogleCloudストレージクラスについて学ぶ

ボリューム階層化ポリシー

階層化するボリュームを選択するときは、各ボリュームに適用するボリューム階層化ポリシーを選択します。階層化ポリシーは、ボリュームのユーザーデータブロックをいつクラウドに移動するか、または移動するかどうかを決定します。

階層化ポリシーなし

データをパフォーマンス層のボリュームに保持し、データがクラウドに移動されないようにします。

コールドスナップショット（スナップショットのみ）

ONTAPは、アクティブなファイルシステムと共有されていないボリューム内のコールドスナップショットブロックをオブジェクトストレージに階層化します。読み取られると、クラウド層のコールドデータブロックがホットになり、パフォーマンス層に移動します。

データは、アグリゲートが50％の容量に達した後、データが冷却期間に達したときにのみ階層化されます。デフォルトの冷却日数は2日ですが、日数は調整できます。

パフォーマンス層の容量が70％を超える場合、クラウド層からパフォーマンス層への書き込みは無効になります。これが発生した場合、ブロックはクラウド層から直接アクセスされます。
コールドユーザーデータ（自動）

ONTAPは、ボリューム内のすべてのコールドブロック（メタデータを含まない）をオブジェクトストレージに階層化します。コールドデータには、Snapshotコピーだけでなく、アクティブなファイルシステムからのコールドユーザーデータも含まれます。

ランダム読み取りで読み取られると、クラウド層のコールドデータブロックがホットになり、パフォーマンス層に移動します。インデックススキャンやウイルス対策スキャンに関連するものなど、順次読み取りによって読み取られる場合、クラウド層のコールドデータブロックはコールドのままであり、パフォーマンス層に書き込まれません。

データは、アグリゲートが50％の容量に達した後、データが冷却期間に達したときにのみ階層化されます。冷却期間は、データが「コールド」と見なされてオブジェクトストアに移動されるために、ボリューム内のユーザーデータが非アクティブのままである必要がある時間です。デフォルトの冷却日数は31日ですが、日数は調整できます。

パフォーマンス層の容量が70％を超える場合、クラウド層からパフォーマンス層への書き込みは無効になります。これが発生した場合、ブロックはクラウド層から直接アクセスされます。
すべてのユーザーデータ（すべて）

すべてのデータ（メタデータを含まない）はすぐにコールドとしてマークされ、できるだけ早くオブジェクトストレージに階層化されます。ボリューム内の新しいブロックが冷えるまで48時間待つ必要はありません。 Allポリシーが設定される前にボリュームに配置されたブロックは、コールドになるまでに48時間かかることに注意してください。

読み取られた場合、クラウド層のコールドデータブロックはコールドのままであり、パフォーマンス層に書き戻されません。このポリシーは、ONTAP9.6以降で使用できます。

この階層化ポリシーを選択する前に、次の点を考慮してください。

  • データを階層化すると、ストレージの効率がすぐに低下します（インラインのみ）。

  • このポリシーは、ボリュームのコールドデータが変更されないことが確実な場合にのみ使用してください。

  • オブジェクトストレージはトランザクションではなく、変更されると大幅な断片化が発生します。

  • データ保護関係のソースボリュームにすべての階層化ポリシーを割り当てる前に、SnapMirror転送の影響を考慮してください。

    データはすぐに階層化されるため、SnapMirrorはパフォーマンス層ではなくクラウド層からデータを読み取ります。これにより、異なる階層化ポリシーを使用している場合でも、SnapMirror操作が遅くなり、キュー内の他のSnapMirror操作が遅くなる可能性があります。

すべてのDPユーザーデータ（バックアップ）

データ保護ボリューム上のすべてのデータ（メタデータを含まない）は、すぐにクラウド層に移動されます。読み取られた場合、クラウド層のコールドデータブロックはコールドのままであり、パフォーマンス層に書き戻されません（ONTAP 9.4以降）。

このポリシーは、ONTAP9.5以前で使用できます。 ONTAP9.6以降のすべての階層化ポリシーに置き換えられました。
